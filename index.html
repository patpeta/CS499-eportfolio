Professional Self-Assessment

Completing the Computer Science program and building this ePortfolio has been a process of growing both technically and professionally.

Over time, coursework has helped develop a stronger understanding of how software is designed, built, and improved. Revisiting earlier projects and enhancing them for this portfolio made it clear how much progress has been made. It also reinforced the important concept that writing code that works is only the starting point, writing code that is clean, maintainable, and scalable is what sets a developer apart.

This program has helped shape what my professional goals should look like in the software engineering field. Improving past work and organizing it into an improved portfolio highlighted the importance of continuous modifications and attention to detail when developing code. These are skills that directly apply to a career in software development, where projects are almost never perfect on the first attempt and often require refinement. This portfolio shows the ability to review existing code to find weaknesses, and make meaningful improvements, which reflects real-world practices within this field.

Collaboration and communication are also important components of being successful in this field. Even when working independently, writing code in a way that others can understand is essential. Clear structure, consistent naming conventions, and useful comments all contribute to making code easier to read and maintain. In addition, the narratives included in this portfolio demonstrate the ability to explain technical decisions and improvements in a clear and organized way.

The program has also strengthened skills in data structures and algorithms. Learning how to organize data efficiently and apply logical problem-solving techniques has improved the ability to design better solutions. Understanding how different approaches impact performance has made it easier to make informed decisions when developing or improving applications.

Software engineering and database development are two areas that have seen significant improvement throughout the program. The ability to design applications with better structure, separate responsibilities within the code, and improve overall maintainability has become a major focus.

Security is another area that has become more important over time. Adding input validation and error handling to applications shows an awareness of how small issues can lead to larger problems if they are not addressed.

Overall, this ePortfolio shows the progress made throughout the Computer Science program. It provides clear examples of ability to move into a professional environment and demonstrates the skills to build and improve software, solve problems using structured approaches, and communicate technical ideas.

Software Engineering

Mobile Inventory Management App

Demonstrates application design, structure, validation, navigation, and maintainability.

Artifact Description

The artifact I selected for this enhancement is a mobile inventory management application developed in Android Studio during the CS 360 Mobile Architecture and Programming course.

Enhancements

Input validation was implemented through a reusable helper class. The application structure was improved by separating responsibilities within the code. Navigation between activities was refined to create a more consistent user experience.

Reflection

Working on these enhancements helped me better understand how important good software design and structure really are. These improvements made the app clearer, more reliable, and easier to use.

Algorithms & Data Structures

Search and Sorting Enhancements

Highlights efficient problem-solving and data organization.

Artifact Description

The artifact used for this enhancement is the same mobile inventory management application that was originally created in Android Studio during the CS 360 course.

Justification for Inclusion

This artifact was chosen because it had room for improvement in how it handled data. Originally, there was no real system in place for organizing, searching, or sorting inventory items.

Enhancements Implemented

A structured collection was added to store items more effectively. Search functionality was added so users can quickly find specific items, and sorting was added to organize items alphabetically.

Course Outcomes Alignment

This enhancement supports the course outcomes related to designing and improving computing solutions using algorithmic thinking.

Reflection

This enhancement improved both the functionality and usability of the application. It shows a better understanding of how algorithms and data structures can be used in real applications.
