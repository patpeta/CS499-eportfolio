Artifacts

Software Engineering

Demonstrates application design, structure, and maintainability.

Artifact Description

The artifact I selected for this enhancement is a mobile inventory management application developed in Android Studio during the CS 360 Mobile Architecture and Programming course I took last year. The application was originally designed to allow users to navigate between screens and input inventory data. While the core structure of the application was somewhat functional, several areas lacked proper organization, validation, and maintainability.

Inclusion Explanation

I selected this artifact because this was a program that I couldn’t get to run when I was taking the mobile architecture class. By revising this artifact, I can demonstrate improved foundational skills in mobile application development and improvements in software design and engineering. The original implementation contained tightly coupled logic within activity classes and limited input validation. Enhancing this artifact allows for the demonstration of key software engineering principles, including modular design, code reusability, and improved user interaction. The artifact shows the ability to analyze an existing codebase, identify weaknesses, and implement structured improvements aligned with industry best practices.

Enhancements

This first set of enhancements of the application highlights several key improvements focused on software design and maintainability. First, input validation was implemented through a reusable helper class. This reduces repeated logic across activities and ensures consistent validation of user input. The introduction of this component improves both reliability and usability. Second, the application structure was improved by separating responsibilities within the code. Activities were modified to focus primarily on user interface interactions, while validation logic was moved to dedicated methods and helper classes. This separation improves maintainability and scalability. Third, navigation between activities was refined to create a more consistent and predictable user experience. Clear transitions between screens improve usability and align with expected mobile application behavior. The login layout page was removed as it shared the same task as the main page and the manifest was updated to navigate using the main XML instead of the login XML. This overall improved navigation. Finally, comments and documentation were added to clarify the purpose of key sections of code. This improves readability and supports collaboration by making the code easier to understand for other developers.

Outcomes

The improvements I made in this milestone connect directly to several course outcomes. By reorganizing the app and cleaning up the structure, I was able to follow better software engineering practices and make the code easier to work with. It also allowed the app to actually run without crashing which was a major win. Breaking things into smaller, more modular pieces and adding input validation helped improve both the quality and maintainability of the code. I also strengthened the app’s security by making sure user input is checked properly and by adding basic error handling to prevent invalid or harmful data from affecting the app.

Process Reflection

Working on these enhancements helped me better understand how important good software design and structure really are. One of the biggest takeaways was how much separating responsibilities instead of putting everything in one place makes the code easier to read, manage, and expand later. One challenge I faced was figuring out how to reorganize the code without breaking anything. The original version had a lot of different responsibilities mixed, so refactoring it required careful changes to keep everything working correctly. Another challenge was deciding how much to abstract the code, especially when creating helper methods and validation logic. I had to find a balance between keeping things simple and making the code reusable. Overall, these improvements made the app clearer, more reliable, and easier to use. The updated structure also feels much closer to what you would see in a real-world development environment and sets a strong foundation for future improvements.

Algorithms & Data Structures

Highlights efficient problem-solving and data organization.

Artifact Description

The artifact used for this enhancement is the same mobile inventory management application that was originally created in Android Studio during the CS 360 course. The app allows users to add and manage inventory items, but in its original version, the way data was handled was very basic and not well organized.

Justification for Inclusion

This artifact was chosen for this enhancement concept because it had a lot of room for improvement in how it handled data. Originally, there was no real system in place for organizing, searching, or sorting inventory items. Everything was handled in a simple way that worked for small amounts of data but would not scale well. This made it a good choice for showing skills related to algorithms and data structures. By improving how the app stores and processes data, it shows a better understanding of how to build more efficient and user-friendly applications.

Enhancements Implemented

The goal of this enhancement was improving how inventory data is organized and accessed. A structured collection was added to store items more effectively, which made it easier to manage and work with the data. Search functionality was added so users can quickly find specific items instead of manually looking through everything. This was done by looping through the collection and returning items that match the search input. In addition, sorting was added to organize items in a consistent way, such as alphabetically, which improves the overall user experience. These changes make the app easier to use and more efficient. Instead of relying on basic input and display, the app now has a more structured way of handling data. This shows how applying simple algorithms and better data organization can make a noticeable difference in how an application performs.

Course Outcomes Alignment

This enhancement supports the course outcomes related to designing and improving computing solutions using algorithmic thinking. Adding search and sorting features shows the ability to apply problem-solving techniques and improve how data is handled. It also demonstrates an understanding of choosing appropriate data structures and making decisions that balance simplicity and performance. These changes reflect how even small improvements in logic can make an application more effective.

Reflection on the Enhancement Process

Working on this part of the project made it clear how important it is to organize data properly. One of the biggest takeaways was how much easier an application becomes to use once it includes basic features like search and sorting. One challenge was figuring out how to structure the data in a way that would support these features without making the code overly complicated. Since the original version didn’t separate data handling very well, it took some effort to adjust the structure while keeping everything working. Another challenge was making sure the new features fit naturally into the app without breaking existing functionality. This required careful testing and small adjustments along the way. Overall, this enhancement improved both the functionality and usability of the application. It shows a better understanding of how algorithms and data structures can be used in real applications, even at a basic level.
